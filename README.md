<details>
<summary><b>Predictive Analytics Dashboard</b> — End-to-End ML Pipeline with Business Intelligence</summary>

<br/>

A production-grade predictive analytics system built to ingest, process, and model structured business data, exposing results through an interactive Streamlit dashboard for non-technical stakeholders.

| Attribute | Details |
|:---|:---|
| **Stack** | Python · Scikit-Learn · Pandas · Streamlit · Power BI · MySQL |
| **Scale** | Processed 100K+ row datasets with automated preprocessing pipelines |
| **Performance** | Achieved 92% classification accuracy with XGBoost ensemble |
| **Security** | Environment-based config management, parameterised SQL queries |
| **Impact** | Reduced manual reporting time by 70%; dashboard adopted by team for weekly KPI reviews |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-0f0c29?style=flat-square&logo=github&logoColor=7C4DFF)](https://github.com/Akshay758) |

</details>

<details>
<summary><b>Sentiment Analysis Engine</b> — NLP Classification System for Textual Data</summary>

<br/>

A deep learning-powered NLP pipeline designed to classify sentiment across large-scale text corpora, incorporating both traditional ML baselines and transformer-based fine-tuning for performance benchmarking.

| Attribute | Details |
|:---|:---|
| **Stack** | Python · TensorFlow · Keras · NLTK · Transformers (HuggingFace) · Pandas |
| **Scale** | Trained on 50K+ labelled reviews with multi-class sentiment labels |
| **Performance** | LSTM model achieved 88% accuracy; BERT fine-tune pushed to 94% |
| **Security** | Sanitised input preprocessing to prevent injection via text inputs |
| **Impact** | Prototype adopted for product feedback classification use-case |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-0f0c29?style=flat-square&logo=github&logoColor=7C4DFF)](https://github.com/Akshay758) |

The system benchmarks three architectures — TF-IDF + Logistic Regression, LSTM with word embeddings, and BERT fine-tuning — enabling a clear performance-cost trade-off analysis for production deployment decisions.

</details>

<details>
<summary><b>Customer Churn Prediction Model</b> — Business Intelligence & Retention Analytics</summary>

<br/>

A churn prediction system built for telecom dataset analysis, combining interpretable ML models with SHAP-based explainability to deliver not just predictions but actionable retention strategies.

| Attribute | Details |
|:---|:---|
| **Stack** | Python · Scikit-Learn · XGBoost · SHAP · Matplotlib · Seaborn · Streamlit |
| **Scale** | Trained on 7,000+ customer records with 20+ engineered features |
| **Performance** | F1-Score of 0.89 on held-out test set with calibrated probability outputs |
| **Security** | No PII stored; dataset anonymised prior to modelling |
| **Impact** | Feature importance analysis identified top 5 churn drivers for business action |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-0f0c29?style=flat-square&logo=github&logoColor=7C4DFF)](https://github.com/Akshay758) |

SHAP waterfall and summary plots were embedded in the Streamlit app, enabling non-technical users to understand which features drove individual customer churn predictions — bridging the gap between ML engineering and business decision-making.

</details>
